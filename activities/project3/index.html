About the project

For this, we were tasked to redesign a website that we believed was in need. As a team, we instantly thought of Yahoo.com. This search engine is notoriously known to be super busy, and almost “spam” looking. The design of this user interface is a hindrance upon users attempting to utilize this platform–which is a huge problem from a business perspective. Our goal was to create an innovative redesign that appealed to today's user. We attempted a sleek, modern approach. We researched other competitors to create a nexus of the best features to display news effectively, while still emphasizing the search engine abilities.

 

Design work

For our redesigning, we found it to be important to get input from different demographics. We used the following methods to base our designs: Survey, User Interviews, System usability scale, and Observational study.

 

Interface in detail:

In order to create a more youthful, modern User Interface we utilized our research. Based on our survey, user interviews and feedback, and personal research of other popular news sites, we were able to create a well-thought-out, innovative approach to presenting news for Yahoo, while also keeping the original feel of the website. We kept the large banner to highlight the main top stories, similar to the original design. However, we opted for a more sleek feel.

 

 

Features and controls

Our redesign project focused on enhancing how articles and content are presented on Yahoo. A significant change was introducing a collage-style layout for article display, inspired by platforms like Pinterest and Instagram, to make the feed more engaging and user-friendly. The current Yahoo layout often feels cluttered, with too many articles competing for attention, but our approach ensures a visually appealing experience while improving content organization.

 

We also revamped the Stories for You feature, which was previously lost in the chaos of the page. We gave it its own dedicated section with a carousel design, strategically placed at the bottom center of the page to ensure it stands out.

 

For the top center of the page, we replaced the large ad banner with a scrollable header and added a right sidebar. The sidebar allows users to customize their experience by selecting up to two features from horoscope, weather, and trending now.

 

Additionally, we increased the size of the search bar for better visibility and usability. To declutter the navigation bar, we moved the categories to a left-side sidebar, enabling a cleaner design that emphasizes the search bar and sidebar elements separately.

Libraries & code structure

When implementing the code for our web application we split the work into different sections from our Figma and each made our own component so it would be easier to glue them all together when merging to the main. We imported two libraries, one was Siema and the other was Svelte-carousel, both were used for implementing different carousels in our website.

 

Use of AI

We used ChatGPT to debug our code. One example of how I used it was when I was having issues implementing the library svelte-carousel I was able to paste some of the GitHub documentation description into the prompt and then asked what is wrong with this small code block based on the provided documentation.

 

Future work

If we had more time there are many features that we would love to work on. Right now, our website isn’t responsive when we scale down the window. This is because not everyone took that into consideration when building our components and caused issues when merging them together–it caused our whole webpage to look off when the display screen changes. One other feature that would add more functionality is getting the side bar icons to filter the articles.
